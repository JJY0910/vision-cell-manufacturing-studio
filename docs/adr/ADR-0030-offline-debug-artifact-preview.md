# ADR-0030: Offline Debug Artifact Preview And Re-inspect Preparation

## Status

Accepted

## Context

Offline Debug can list persisted inspection results and report artifact metadata state. Priority 1 now requires artifact viewing and a Re-inspect preparation structure while keeping actual equipment execution out of scope.

Artifact viewing must not read files directly from WPF. The preview path should reuse the Application/Persistence artifact reader boundary and stay limited to the deterministic BMP evidence files currently generated by the simulator inspection pipeline.

## Decision

- Extend `IInspectionArtifactReader` with `ReadPreviewAsync`.
- Add `InspectionArtifactPreviewResult` and `InspectionArtifactPreviewPixelFormat` to carry WPF-independent BGRA preview pixels.
- Implement safe BMP preview decoding in `FileSystemInspectionArtifactWriter` for relative artifact paths under the configured artifact root.
- Add Offline Debug commands for loading selected overlay/height-map previews and preparing a Re-inspect context.
- Add `InspectionReinspectPreparation` as a structure-only Application record. It does not execute inspection.
- Bind overlay and height-map preview images plus Re-inspect prepared status in `OfflineDebugView`.

## Alternatives Considered

- Let WPF load BMP files directly: rejected because WPF should not own file-system-specific artifact access.
- Open the local file path with the OS shell: rejected for this slice because safe file launching needs a separate operator confirmation and path policy.
- Execute Re-inspect immediately: rejected because actual re-inspection requires sequence, Recipe, lot, and equipment state policy that must be implemented separately.

## Consequences

- Offline Debug can render deterministic overlay and height-map BMP artifacts as preview images.
- Artifact preview failures reuse not-recorded, missing, unsafe, and unavailable states.
- Re-inspect now has a typed preparation record without claiming production re-run readiness.
- The preview decoder currently supports uncompressed 24-bit BMP files generated by the project artifact writer.

## Requirement Coverage

- FR-200/FR-220/FR-221/FR-222: Offline Debug can list result records, view overlay/height-map artifacts, and prepare a re-inspection context.
- FR-260: Inspection evidence remains visible for support/debug workflow review.
- NFR-001/NFR-002: WPF stays MVVM-only and receives WPF-independent preview pixels from Application/Persistence boundaries.
- NFR-004/NFR-006: Preview load failures remain operator-visible.
- NFR-SEC-001: Preview reads use the same relative path and traversal rejection policy as artifact metadata.
- NFR-TEST-001: Persistence and App tests cover preview loading and Re-inspect preparation.

## Rollback

Remove `ReadPreviewAsync`, preview result records, `InspectionReinspectPreparation`, Offline Debug preview/Re-inspect bindings, the new tests, this ADR, and related documentation updates.
